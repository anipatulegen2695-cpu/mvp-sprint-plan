# Git стратегия

## Main branch
Main — дайын код сақталады

## Коммит форматы
- feat: жаңа функция
- fix: қате түзету
- docs: құжат
- design: дизайн

## Feature branches
feature/test-page сияқты бөлек branch

## Commit жасау
Әр маңызды өзгерістен кейін commit

## Pull Request
Feature → Main қосу үшін тексеру

## Академиялық адалдық
Git history арқылы кім не жасағаны көрінеді, сондықтан көшірме емес, өз жұмысымыз дәлелденеді.
